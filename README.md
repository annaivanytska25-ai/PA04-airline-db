# PA04-airline-db

### Опис бази даних
Створено базу даних для авіакомпанії. Вона моделює основні бізнес-процеси : бронювання місця, призначення
літаків в аеропорти, призначення членів екіпажу до літака, та інші. Дані були згенеровані за допомогою faker.

### Структура
Було створено 9 таблиць : passengers - контактні дані пасажирів, passenger_details - додаткові дані, такі
як громадянство та статус лояльності, aircraft - літаки компанії ( модель та кількість місць ), airports - аеропорти та їх дані,
flights - рейси та їх дані, seats - місця в літаках ( номер та клас ), bookings - бронювання пасажирів, crew_members - 
члени екіпажу та flight_crew - призначення екіпажу до рейсів.

### Звʼязки між таблицями
- 1:1 - пасажири та їхні деталі ( passenger_details );
- 1:many - один літак на багато рейсів, один літак має багато місць;
- 1:many - аеропорт є або точкою А або Б в залежності від рейсу, у пасажира може бути багато бронювань;
- many:many - рейси та екіпаж літака ( призначення екіпажу до рейсу в flight_crew );

### Введені обмеження
- primary key на кожній таблиці
- foreign key з on delete cascade для залежних таблиць. Із системи видаляється пасажир - із ним його деталі;
- unique на електронній пошті, номері паспорта, та інші;
- not null для усіх обовʼязкових полів ( персональні дані пасажира, дані аеропорту, та інші );
- check для перевірок, наприклад, кількість місць має обовʼязково бути > 0;
- default для статусу рейсів та бронювань;

### Explain Analyze
Був проведений тестовий запит для порівняння планів виконання без та із індексом. 
Query plan показав перехід від повного сканування таблиці seq scan до індексного сканування index scan.
Час виконання зменшився у ~26 разів.

## Додаткові завдання 

### View
Було створене представлення, яке відображає всі активні рейси із статусом 'scheduled' або 'boarding'.
Має в собі ID та номер рейсу, модель літака, призначена на конкретний рейс, дані про відправлення та прибуття,
та статус рейсів. Такий набір даних є дуже зручним для моніторингу активних рейсів.
